Cambios en la version 4.0.0-beta

  • Soporta el modelo de aplicación de Levantamiento Catastral v1.2.
  • Informalidad:
    • Se introduce simbología para identificar la informalidad en terrenos, construcciones y unidades de construcción.
    • Los predios informales se pueden recorrer uno a uno en el módulo de consultas, obteniendo la información de los mismos por componente.
  • Refactor, mejoras y optimización del módulo de reglas de calidad:
    • El núcleo (core) del módulo ha sido reescrito para darle más flexibilidad y compatibilidad con Add-ons.
    • Se utiliza un modelo de INTERLIS de errores de calidad para almacenar los errores de calidad encontrados.
    • Se utilizan catálogos de INTERLIS (listados de reglas de calidad en .XTF) para no tener que modificar el modelo si se agregan o eliminan reglas de calidad.
    • La validación de iliValidator es ahora una regla de calidad, y se comporta como las demás: los resultados se guardan en la misma base de datos y aparecen en el reporte en PDF de la validación de reglas de calidad.
    • La interfaz gráfica fue rediseñada. Ahora se usa un panel (dockwidget) que permite buscar reglas y analizar individualmente los errores encontrados.
    • La regla de calidad "Los terrenos no deben superponerse" soporta informalidad. Esto es, si dos terrenos se superponen pero uno es formal y el otro informal, no se considera como error topológico.
  • Se introduce el soporte de Add-ons para extender la funcionalidad del Asistente LADM-COL. Los Add-ons son plugins de QGIS que dependen del Asistente LADM-COL. Por ejemplo, el Add-on de Ambiente agrega varios modelos, un rol y nuevas funcionalidades relativas al sector Ambiente.
  • En importación masiva de datos, los mapeos de campos muestran los campos obligatorios resaltados en color naranja para facilitar el llenado.
  • El orden de los atributos de cada formulario es configurable en el archivo LADM_COL_configuration.toml. (Colaboración de Jonny Sánchez)
  • Refactor del módulo de wizards para mejorar su mantenimiento. Es un refactor a nivel de núcleo (core) y no de interfaz.
  • Se elimina la dependencia con el plugin QGIS Model Baker. El usuario no necesitará instalar dicho plugin pues se ha incluido como librería dentro del Asistente LADM-COL.
